페이지와 라우팅 - Vue Router

1SPA(Single Page Application)에서 페이지를 이동하면 무슨 일이 일어나나요?

서버에서 새로운 HTML 파일을 받아와서 전체 페이지가 새로고침된다
HTML은 그대로이고, 화면의 일부(RouterView)만 바뀐다
브라우저가 완전히 닫혔다가 다시 열린다
JavaScript가 비활성화되고 순수 HTML로 전환된다
정답! SPA는 하나의 HTML 페이지 안에서 RouterView 부분만 교체합니다. 그래서 깜빡임 없이 부드럽게 화면이 전환되죠.
오답! SPA는 Single Page Application — 하나의 페이지 안에서 필요한 부분(RouterView)만 바꿉니다. 서버에 새 HTML을 요청하지 않아요.

2<RouterView />의 역할은?

<template>
  <nav>...메뉴...</nav>
  <main>
    <RouterView />
  </main>
</template>
메뉴(nav)를 화면에 표시한다
페이지를 새로고침한다
현재 URL에 맞는 컴포넌트를 여기에 표시한다
모든 라우트의 컴포넌트를 동시에 표시한다
정답! RouterView는 액자 틀 같은 역할입니다. URL이 /login이면 LoginView를, /boards면 BoardsView를 그 자리에 표시합니다.
오답! RouterView는 현재 URL에 해당하는 컴포넌트를 표시하는 자리입니다. URL이 바뀌면 이 자리의 내용만 교체되고, nav 같은 주변 요소는 그대로 유지됩니다.

3<RouterLink>와 일반 <a> 태그의 차이는?

RouterLink는 페이지 새로고침 없이 이동하고, a 태그는 전체 페이지를 새로 불러온다
RouterLink는 외부 사이트로 이동하고, a 태그는 내부 페이지로 이동한다
둘 다 완전히 동일하게 동작한다
RouterLink는 스타일이 자동 적용되고, a 태그는 스타일이 없다
정답! RouterLink는 Vue Router를 통해 클라이언트 측에서 URL만 바꾸고 컴포넌트를 교체합니다. a 태그는 브라우저가 서버에 새 HTML을 요청하여 전체 페이지를 다시 불러옵니다.
오답! 핵심 차이는 페이지 새로고침 여부입니다. RouterLink는 SPA 방식으로 화면의 일부만 바꾸고, a 태그는 전체 페이지를 새로 불러옵니다.

4URL이 /boards/42일 때, 42를 컴포넌트에서 가져오는 방법은?

// router/index.ts
{ path: '/boards/:id', component: BoardDetailView }
req.params.id
this.$id
window.location.id
route.params.id (useRoute() 사용)
정답! Vue에서는 useRoute()로 현재 라우트 정보를 가져오고, route.params.id로 동적 라우트의 값에 접근합니다. req.params.id는 Express(백엔드)에서 쓰는 방식이에요.
오답! req.params.id는 Express(백엔드)에서 쓰는 방식입니다. Vue(프론트엔드)에서는 useRoute()를 호출하고 route.params.id로 접근합니다.

5다음 라우터 설정에서 URL /boards/7에 매칭되는 라우트는?

routes: [
  { path: '/', component: HomeView },
  { path: '/boards', component: BoardsView },
  { path: '/boards/:id', component: BoardDetailView },
  { path: '/login', component: LoginView },
]
path: '/boards' → BoardsView
path: '/boards/:id' → BoardDetailView (id = "7")
path: '/' → HomeView
매칭되는 라우트가 없어서 에러 발생
정답! /boards/7은 /boards/:id 패턴과 매칭되고, :id에 "7"이 들어갑니다. /boards는 정확히 /boards일 때만 매칭됩니다.
오답! /boards는 정확히 /boards일 때만 매칭됩니다. /boards/7은 /boards/:id 패턴과 매칭되어 BoardDetailView가 표시되고, id 파라미터에 "7"이 들어갑니다.
