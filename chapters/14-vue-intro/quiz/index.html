Chapter 14 Quiz

Vue.js 첫 걸음 - 컴포넌트와 반응성

1Vue의 .vue 파일은 어떤 3가지 영역으로 구성되나요?

HTML, CSS, JSON
<script setup>, <template>, <style>
<head>, <body>, <footer>
import, export, render
정답! script(로직/두뇌), template(화면/몸), style(스타일/옷) — 하나의 .vue 파일 안에 모두 들어있는 것이 컴포넌트입니다.
오답! Vue 컴포넌트(.vue 파일)는 <script setup>(로직), <template>(화면), <style>(스타일) 3개 영역으로 구성됩니다.

2ref()로 선언한 데이터의 특징은?

const count = ref(0)
count.value++  // 값을 바꾸면?
값을 바꿔도 화면은 변하지 않는다
값을 바꾸려면 반드시 setState()를 호출해야 한다
한 번 선언하면 값을 바꿀 수 없다 (읽기 전용)
값이 바뀌면 화면이 자동으로 갱신된다
정답! 이것이 Vue의 핵심인 반응성(reactivity)입니다. ref()로 만든 데이터는 값이 바뀌면 해당 데이터를 사용하는 화면이 자동으로 갱신됩니다.
오답! ref()는 반응성 데이터를 만듭니다. 값이 바뀌면 Vue가 감지해서 화면을 자동으로 갱신해주는 것이 핵심이에요.

3v-model의 역할은?

<input v-model="name" />
<p>{{ name }}</p>
입력칸과 데이터를 양방향으로 연결 — 입력하면 데이터가 바뀌고, 데이터가 바뀌면 입력칸도 바뀐다
입력칸의 스타일을 설정한다
입력칸을 읽기 전용으로 만든다
입력값을 서버로 자동 전송한다
정답! v-model은 양방향 바인딩입니다. 사용자가 입력하면 name 변수가 바뀌고, 코드에서 name을 바꾸면 입력칸의 내용도 바뀝니다.
오답! v-model은 양방향 바인딩(two-way binding)으로, 입력칸과 데이터를 양쪽으로 연결합니다. 아까 이름을 입력하면 바로 화면에 반영된 것이 이 기능이에요.

4게시글 목록을 화면에 반복해서 보여주려면 어떤 디렉티브를 사용하나요?

// posts = [{ title: "첫 글" }, { title: "둘째 글" }]
// 각 글의 제목을 li 태그로 반복 렌더링하고 싶다면?
v-if="posts"
v-model="posts"
v-for="post in posts"
v-bind="posts"
정답! v-for는 배열을 반복해서 HTML 요소를 생성합니다. "posts 배열에서 하나씩 꺼내서 post라는 이름으로 쓰겠다"는 뜻이에요.
오답! v-if는 조건부 표시, v-model은 양방향 바인딩입니다. 배열을 반복 렌더링할 때는 v-for="post in posts"를 사용합니다.

5다음 코드에서 버튼을 3번 클릭하면 화면에 표시되는 숫자는?

<script setup>
import { ref } from 'vue'
const count = ref(10)
</script>

<template>
  <button @click="count += 5">{{ count }}</button>
</template>
15
25
10
30
정답! 초기값 10에서 클릭할 때마다 5씩 증가합니다. 10 → 15 → 20 → 25. ref의 반응성 덕분에 매 클릭마다 화면이 자동 갱신됩니다.
오답! count의 초기값은 10이고, @click="count += 5"이므로 클릭할 때마다 5가 더해집니다. 3번 클릭: 10 → 15 → 20 → 25입니다.
