Q1. 서버와 브라우저의 관계를 식당에 비유하면?

???
→ 요청 →
???
→ 응답 →
???
① 주방 → 메뉴판 → 손님
② 손님(브라우저) → 주방(서버) → 손님(브라우저)
③ 서버 → 브라우저 → 서버
④ Express → Node.js → 브라우저
정답! 브라우저(손님)가 요청을 보내면, 서버(주방)가 처리해서 브라우저에게 응답을 돌려줍니다.
다시 생각해보세요. 누가 먼저 요청을 보내나요?

Q2. 아래 코드에서 빈칸에 들어갈 것은?

const express = require('express');
const app = express();

app.get('/', (req, ???) => {
  ???.send('Hello!');
});
① req, req
② app, app
③ res, res
④ express, express
정답! res는 응답(response) 객체예요. res.send()로 클라이언트에게 응답을 보냅니다.
힌트: 응답을 보내는 건 request일까요, response일까요?

Q3. app.get('/about', ...) 의 의미는?

app.get('/about', (req, res) => {
  res.send('소개 페이지');
});
① /about 주소로 요청이 오면 '소개 페이지'를 응답한다
② about이라는 변수를 가져온다
③ 서버를 about 모드로 실행한다
④ /about 폴더에 파일을 저장한다
정답! app.get(경로, 함수)는 해당 경로로 GET 요청이 오면 함수를 실행합니다.
식당 비유를 떠올려보세요. 메뉴판에 메뉴를 등록하는 것과 같아요.

Q4. app.listen(3000)은 무슨 뜻일까요?

① 3000개의 요청만 받겠다
② 3000초 후에 서버를 종료한다
③ 3000번째 라우트를 실행한다
④ 3000번 포트에서 요청을 기다린다
정답! listen(포트)는 해당 포트 번호에서 요청을 기다리라는 뜻입니다. 그래서 http://localhost:3000 으로 접속할 수 있는 거예요.
포트(port)는 서버의 문 번호라고 생각하면 돼요. 어떤 문에서 기다릴지 정하는 겁니다.

Q5. Express 없이도 서버를 만들 수 있을까요?

① 아니요, Express가 없으면 서버를 만들 수 없다
② 네, Node.js만으로도 가능하지만 Express가 더 편하다
③ 네, 브라우저만으로 서버를 만들 수 있다
④ 아니요, npm이 서버를 만든다
정답! Node.js가 빈 주방이라면, Express는 편리한 도구가 갖춰진 주방이에요. 빈 주방에서도 요리는 가능하지만 훨씬 불편하죠.
챕터 초반의 설명을 떠올려보세요. Express는 어떤 역할이었나요?
