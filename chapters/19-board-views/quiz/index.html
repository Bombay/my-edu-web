1다음 두 코드의 차이는?

<RouterLink to="/boards">게시판</RouterLink>
<RouterLink :to="`/boards/${board.id}`">{{ board.name }}</RouterLink>
첫 번째는 버튼이고 두 번째는 링크다
콜론(:)이 붙으면 값을 JavaScript 표현식으로 해석하므로, 두 번째는 board.id 값에 따라 동적으로 URL이 생성된다
두 번째는 문법 오류로 작동하지 않는다
첫 번째는 SPA, 두 번째는 전체 새로고침이 일어난다
정답! 콜론(:)은 v-bind의 축약형이에요. 붙으면 값을 JavaScript로 해석하므로 템플릿 리터럴 같은 표현식을 쓸 수 있어요. 붙지 않으면 그냥 문자열 리터럴.
오답! 콜론(:)은 속성을 동적으로 바인딩하는 v-bind의 축약형이에요. 붙이면 JS 표현식으로, 안 붙이면 문자열 리터럴로 해석돼요. :to, :src, :disabled 등 어떤 속성에도 적용 가능합니다.

2게시판 생성 후 boards.value.push(newBoard)를 쓰는 이유는?

서버에 데이터가 저장되지 않으므로 프론트에서라도 보여주려고
Pinia에 저장하기 위한 필수 문법이다
서버에서 목록을 다시 fetch하지 않고도 Vue의 반응성으로 화면을 즉시 갱신할 수 있어서 (서버 요청 절약 + 빠른 UX)
localStorage에 저장하기 위해서
정답! 서버가 방금 만든 게시판 객체를 돌려주니까, 그걸 배열에 push만 하면 Vue가 자동으로 <li>를 하나 더 추가해요. GET /api/boards를 또 호출할 필요가 없어서 효율적이에요.
오답! 서버는 이미 데이터를 저장했고 새 게시판 객체를 응답으로 돌려줬어요. 그걸 배열에 push만 하면 Vue 반응성 덕분에 화면이 즉시 갱신돼요. 서버에 불필요한 GET 요청을 또 보낼 필요가 없으니 효율적이에요.

3게시판 생성 코드에는 토큰 관련 코드가 전혀 없는데 서버 인증은 어떻게 통과될까요?

await api('/boards', {
  method: 'POST',
  body: JSON.stringify({ name, description })
})
// 어디에도 token을 직접 다루는 코드가 없음!
POST 요청은 토큰 없이도 통과된다
Vue가 자동으로 토큰을 첨부한다
브라우저가 fetch에 자동으로 토큰을 포함시킨다
Ch.18에서 api 모듈을 수정해, localStorage에서 토큰을 읽어 Authorization 헤더에 자동 첨부하도록 만들어뒀기 때문
정답! api 모듈이 중앙에서 토큰을 붙여주니까, 각 컴포넌트는 토큰을 의식하지 않고도 인증된 요청을 보낼 수 있어요. 이게 "공통 로직을 한 곳에 모으기"의 힘이에요.
오답! Vue도, 브라우저도 토큰을 자동으로 붙여주지 않아요. Ch.18에서 api 모듈 안에 localStorage.getItem('token')으로 토큰을 꺼내 Authorization: Bearer ${token} 헤더를 추가하는 로직을 넣어뒀기 때문이에요.

4로그인한 사용자에게만 생성 폼이 보이게 하려면?

라우트 가드를 추가한다
CSS로 display: none 처리한다
authStore.isLoggedIn을 v-if로 확인하여 폼 섹션을 조건부 렌더링한다
서버에서 HTML을 다르게 내려준다
정답! 라우트 가드는 페이지 접근 자체를 막는 용도고, 같은 페이지 안에서 일부 UI를 숨기는 건 v-if가 적합해요. authStore.isLoggedIn은 computed라 로그인 상태가 바뀌면 자동으로 UI가 반응해요.
오답! 라우트 가드는 페이지 "접근" 차단용이에요. 페이지 안에서 특정 UI만 조건부로 보이려면 v-if가 적합합니다. authStore.isLoggedIn은 computed라 반응형으로 동작해서 로그아웃하면 폼이 자동으로 사라져요.

5이번 챕터에 새로운 Vue 개념이 거의 없었던 이유는?

게시판 기능이 단순해서 새 기술이 필요 없기 때문
이전 챕터들에서 배운 v-if, v-model, api 모듈, authStore, try/catch 등을 조합하여 실전 기능을 구성하는 단계이기 때문
Vue는 더 이상 배울 게 없기 때문
선생님이 가르칠 내용을 빼먹었기 때문
정답! 프레임워크 학습의 특징이에요. 초반엔 개념이 많이 쏟아지지만, 어느 시점부터는 배운 것들을 "조합"해서 실전 기능을 만드는 단계로 넘어가요. 지금이 그 전환점이에요.
오답! Vue에는 아직 더 많은 기능이 있지만, 커뮤니티 게시판을 만들기엔 지금까지 배운 것들로 충분해요. 이번 챕터는 "조합의 단계" — 쌓아온 조각들로 실전 기능을 만드는 시점이에요.
