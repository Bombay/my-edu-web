Chapter 21 Quiz

UI 다듬기 — 전역 CSS · CSS 변수 · 미디어쿼리 · 스코프 분리

1main.ts에 import './assets/main.css' 한 줄을 추가했더니 모든 컴포넌트에서 .card 클래스를 쓸 수 있게 됐습니다. 이게 가능한 이유는?

Vue가 자동으로 모든 .vue 파일을 스캔해서 클래스를 등록한다
main.ts에서 import한 모든 변수는 컴포넌트에 자동 주입된다
Vite가 import된 CSS를 빌드 시 하나의 스타일시트로 묶어 페이지에 주입하므로, 같은 페이지의 모든 컴포넌트가 그 규칙을 공유함
CSS 파일은 어디서나 자동으로 읽혀서 import가 사실 필요 없다
정답! Vite/Webpack 같은 번들러가 import된 CSS를 모아 하나의 시트로 만들고 <head>에 주입해요. 그래서 어떤 컴포넌트에서든 같은 클래스 이름을 쓸 수 있는 거예요. 이게 "전역 CSS"가 동작하는 원리.
오답! Vue는 .vue 파일을 자동 스캔하지 않고, JS 변수 import와 CSS는 다르게 동작해요. 핵심은 번들러(Vite)가 import된 CSS를 하나의 스타일시트로 묶어 페이지에 주입한다는 점입니다. 그래서 모든 컴포넌트가 같은 클래스를 공유할 수 있어요.

2다음 코드의 의도와 효과는?

:root {
  --color-primary: #3b82f6;
}
.btn { background: var(--color-primary); }
.list a { color: var(--color-primary); }
버튼과 링크 색을 다르게 만들기 위해
색을 한 곳(변수 선언)에서 관리하여, 한 줄만 바꾸면 그 변수를 쓰는 모든 요소의 색이 함께 바뀌게 하기 위해
JavaScript에서 색을 동적으로 바꾸기 위해 필수로 필요한 문법
CSS 성능을 향상시키는 최적화 기법
정답! CSS 변수(--color-primary)는 디자인 토큰의 핵심 도구예요. 한 곳만 바꾸면 그 변수를 쓰는 모든 요소가 자동 반영. "테마 색 살짝 어둡게" 한 마디를 5초만에 처리할 수 있어요.
오답! :root에 정의한 CSS 변수는 디자인 토큰 역할이에요. 변수 선언 한 줄만 바꾸면 var(--color-primary)를 쓰는 모든 곳의 색이 일괄 변경됩니다. JS 없이 CSS만으로 가능하고, 성능 최적화와는 무관해요.

3이 미디어쿼리는 어떤 화면에서 활성화되나요?

.btn { width: auto; }

@media (max-width: 768px) {
  .btn { width: 100%; }
}
모든 화면에서 항상 width: 100%가 적용된다
화면이 768px 이상일 때 width: 100%가 적용된다
화면이 768px 이하일 때만 width: 100%가 적용되고, 그보다 넓으면 width: auto가 적용된다
미디어쿼리는 데스크탑 브라우저에선 무시된다
정답! @media (max-width: 768px)는 "최대 768px까지" — 즉 768px 이하인 좁은 화면에서만 활성화돼요. 평소엔 .btn { width: auto }가 살아있고, 좁아지면 100%로 덮어써집니다. 모바일 우선 사고에서 자주 쓰는 패턴.
오답! max-width는 "최대 너비"라는 조건 — 768px 이하인 화면에서만 안쪽 규칙이 활성화돼요. 평소엔 바깥의 .btn { width: auto }가 적용되고, 좁아지면 width: 100%가 그것을 덮어씁니다. 데스크탑에서도 화면을 좁히면 적용돼요.

4PostDetailView의 본문에 줄바꿈을 살리는 white-space: pre-wrap 스타일을 어디에 두는 것이 가장 적절한가요?

전역 main.css에 .post-content 클래스로 추가한다 — 어차피 한 곳만 쓰니 안 보이게 두자
PostDetailView.vue 안에 <style scoped>로 둔다 — 그 컴포넌트에서만 의미 있는 좁은 스타일이므로
인라인 style로 그대로 둔다 — 어차피 한 줄이니까
JavaScript로 동적으로 적용한다
정답! "전역엔 여러 곳에서 쓰는 것, 스코프엔 그 컴포넌트만의 것" — 이게 좋은 분리 기준이에요. 글 본문 줄바꿈은 PostDetailView에서만 의미 있으니 scoped가 적절. 전역 CSS가 비대해지는 것도 막아줘요.
오답! 한 컴포넌트에서만 의미 있는 좁은 스타일을 전역 CSS에 넣으면 시간이 갈수록 main.css가 비대해지고 어디서 쓰는지 추적하기 어려워져요. 인라인은 문법 강조도 안 되고 재사용도 안 됩니다. <style scoped>가 정답 — Vue가 자동으로 그 컴포넌트에만 적용되는 고유 클래스를 붙여줘요.

5.container { max-width: 800px; margin: 0 auto; }를 <main>에 적용한 효과는?

main 영역이 항상 800px 폭을 차지한다 (화면이 좁아도 800px 유지 → 가로 스크롤)
main이 화면 좌측에 800px 폭으로 붙는다
main의 글자 크기가 800px이 된다
main이 최대 800px까지 커지지만 그보다 좁은 화면에선 화면 너비에 맞춰 줄어들고, 좌우 여백은 자동으로 균등 분배되어 가운데 정렬된다
정답! max-width는 "최대 그 값까지만 커진다"는 의미 — 화면이 좁으면 그에 맞춰 줄어들어요. margin: 0 auto는 좌우 여백을 동일하게 자동 계산 → 가운데 정렬. 이 두 줄 조합이 "콘텐츠 가운데 모으기"의 표준 패턴이에요.
오답! max-width는 "최대" — 항상 그 폭이 아니라, 화면이 좁아지면 함께 줄어들어 가로 스크롤이 안 생겨요. margin: 0 auto는 좌우 여백 자동 분배로 가운데 정렬을 만듭니다. 이 두 줄 조합이 반응형 콘텐츠 정렬의 표준 패턴이에요.
