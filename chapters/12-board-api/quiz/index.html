게시판 API - 카테고리와 페이지네이션

1boards와 posts의 관계는?

boards: 자유게시판, 질문게시판
posts:  글A(board_id:1), 글B(board_id:2), 글C(board_id:1)
N:M — 하나의 글이 여러 게시판에 속할 수 있다
1:N — 하나의 게시판에 여러 글이 속한다
1:1 — 하나의 게시판에 하나의 글만 속한다
N:1 — 여러 게시판이 하나의 글을 공유한다
정답! 하나의 게시판(board)에 여러 글(posts)이 속하는 1:N 관계입니다. posts 테이블의 board_id가 이 관계를 표현합니다.
오답! 자유게시판에 글A, 글C가 속하고, 질문게시판에 글B가 속합니다. 하나의 게시판에 여러 글 — 1:N 관계예요.

23페이지의 글을 가져오려면? (페이지당 10개)

SELECT * FROM posts LIMIT 10 OFFSET 10
SELECT * FROM posts LIMIT 30 OFFSET 0
SELECT * FROM posts LIMIT 10 OFFSET 20
SELECT * FROM posts LIMIT 20 OFFSET 10
정답! offset = (page - 1) * limit = (3 - 1) * 10 = 20. 앞의 20개를 건너뛰고 10개를 가져옵니다. 즉 21~30번째 글이 나옵니다.
오답! 공식은 offset = (page - 1) * limit입니다. 3페이지면 (3-1) * 10 = 20개를 건너뛰고(OFFSET 20), 10개를 가져옵니다(LIMIT 10).

3전체 글이 25개이고 페이지당 10개일 때, 총 페이지 수는?

const totalPages = Math.ceil(total / limit);
// Math.ceil = 올림 함수
2 페이지
2.5 페이지
25 페이지
3 페이지
정답! 25 / 10 = 2.5인데, Math.ceil(올림)하면 3입니다. 마지막 페이지에 5개만 있더라도 1페이지로 쳐야 하니까요.
오답! 25 / 10 = 2.5이고, Math.ceil(올림)하면 3입니다. 1페이지(10개) + 2페이지(10개) + 3페이지(5개) = 총 3페이지가 필요해요.

4기존 posts 테이블에 board_id 컬럼을 추가할 때 사용하는 SQL은?

ALTER TABLE posts ADD COLUMN board_id INTEGER REFERENCES boards(id);
CREATE TABLE posts ADD board_id INTEGER;
UPDATE posts SET board_id = INTEGER;
INSERT INTO posts (board_id) VALUES (INTEGER);
정답! ALTER TABLE은 이미 존재하는 테이블의 구조를 변경합니다. ADD COLUMN으로 새 컬럼을 추가하고, REFERENCES로 외래 키 관계를 설정합니다.
오답! 이미 존재하는 테이블에 컬럼을 추가하려면 ALTER TABLE ... ADD COLUMN을 씁니다. CREATE는 새 테이블 생성, UPDATE는 데이터 수정, INSERT는 데이터 삽입이에요.

5게시판별 글 목록 API의 URL 설계로 가장 적절한 것은?

GET /api/posts?board=1
GET /api/boards/1/posts
GET /api/getPosts/board/1
POST /api/boards/1/list
정답! REST 원칙에서 "1번 게시판에 속한 글 목록"은 /api/boards/1/posts로 표현합니다. 리소스의 계층 관계(게시판 → 글)가 URL에 자연스럽게 드러나죠.
오답! REST에서는 리소스의 계층 관계를 URL로 표현합니다. "1번 게시판의 글 목록"은 /api/boards/1/posts가 가장 자연스럽습니다. GET은 조회, 동사(getPosts)는 URL에 넣지 않아요.
