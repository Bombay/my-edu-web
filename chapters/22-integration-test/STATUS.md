## 핵심 개념
1. **통합 테스트**: 각 부분은 동작하지만 흐름으로 합쳤을 때 생기는 버그를 잡기 위함. "회원가입→로그인" 같은 사용자 시나리오를 끝~끝으로 점검.
2. **JWT 페이로드는 누구나 디코드 가능**: 점(.)으로 3등분된 토큰의 가운데 부분은 Base64 인코딩일 뿐 암호화가 아님 → 비밀번호 같은 민감 정보는 절대 넣지 않음. 위변조 방지는 마지막 "서명"이 담당.
3. **미들웨어 (복습)**: 요청과 라우트 핸들러 사이의 중간 관문. `next()` 호출해야 다음 단계로 진행. 인증 미들웨어가 토큰 검사를 통과시키지 않으면 라우트 핸들러까지 도달 못함.
4. **에러 미들웨어 (복습)**: 매개변수가 **4개**(`err, req, res, next`)인 게 일반 미들웨어와 구분되는 표식. Express가 자동으로 인식. 라우트 핸들러에서 `throw` → `catch` → `next(err)`로 전달하면 한 곳에서 일관된 에러 응답을 만들 수 있음.
5. **통합 테스트가 잡아내는 버그 유형**: 프론트와 백엔드가 합의 안 된 키 이름(컨벤션 불일치) 같은 "사이의 약속" 문제. 단위로는 안 보이고 합쳐서만 보임.
