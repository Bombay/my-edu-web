# 챕터 진행 규칙

## 학습 시작 시 자동 동작
학습자가 "시작", "다음", 챕터 번호, 또는 아무 학습 관련 요청을 하면:
1. `progress.json`을 읽어 현재 상태를 파악한다
2. 현재 상태에 따라 분기:
   - **이전 챕터 미완료** → 이전 챕터 완료 안내 (퀴즈 통과 필수)
   - **다음 챕터 미시작** → 자동으로 다음 챕터 시작, STATUS.md를 `in_progress`로 갱신
   - **현재 챕터 진행 중** → 체크리스트 확인하여 이어서 진행
   - **특정 챕터 번호 지정** → 이전 챕터 모두 완료 확인 후 시작

## 챕터 진행 순서 (매 챕터 필수)
각 단계를 완료할 때마다 STATUS.md를 실시간으로 갱신한다.

1. **동기부여** — 이전 챕터의 한계/불편함 보여주기 → STATUS.md에 기록
2. **개념 소개** — 비유와 함께 짧게 (핵심 1~2개만) → 핵심 개념을 STATUS.md에 기록
3. **따라하기** — 단계별 안내, 학습자가 직접 코드 작성 → 작성한 코드/파일을 STATUS.md에 기록
4. **스스로 해보기** — 변형 과제 → 결과를 STATUS.md에 기록
5. **시각 확인** — ERD/브라우저/API 응답으로 결과 확인
6. **퀴즈 게이트** — HTML 퀴즈 페이지를 생성하여 브라우저에서 풀게 한다

## 게이트 규칙
- 챕터 N+1을 시작하려면 챕터 N의 STATUS.md가 `completed`이고 `quizPassed`가 `true`여야 한다
- 퀴즈 오답 시: 해당 개념을 다시 설명하고 재도전
- 학습자가 정답의 이유를 직접 설명할 수 있어야 통과
- **절대로 이 규칙을 우회하지 않는다**

### 퀴즈 통과 코드 시스템
퀴즈를 통과하면 브라우저에 **통과 코드**가 표시된다. 학습자가 이 코드를 채팅에 붙여넣어야만 통과로 인정한다.
1. 퀴즈 HTML 생성 시, 통과 시 표시할 고유 코드를 `PASS-{챕터번호}-{랜덤4자리}` 형식으로 삽입한다 (예: `PASS-15-K7X2`)
2. 퀴즈 통과 화면에 코드를 크게 표시하고 **"이 코드를 Claude에게 보내주세요"** 안내 문구를 넣는다
3. 학습자가 코드를 붙여넣으면 STATUS.md의 `passCode`와 대조하여 일치 시 통과 처리한다
4. 코드 없이 "통과했어요"라고만 하면 코드를 요청한다 — **코드 없이 통과 처리하지 않는다**
5. Ch.1~14는 이미 완료되어 passCode 없이 통과 인정됨 (호환성). Ch.15부터 코드 시스템 적용

## 퀴즈 형식
- 각 챕터의 `chapters/XX/quiz/index.html`로 생성
- Mermaid 다이어그램 + 코드 스니펫 기반 문제
- 브라우저에서 인터랙티브하게 풀기
- 통과 기준: 전체 문제의 80% 이상 정답
- **통과 시 화면에 고유 코드 표시** — 학습자가 이 코드를 Claude에게 보내야 완료 인정
- 통과 코드는 퀴즈 생성 시 STATUS.md의 `passCode`에도 기록하여 대조 가능하게 한다

## 커밋 규칙
- 커밋 전 반드시 `progress.json`과 해당 챕터 `STATUS.md`를 최신 상태로 갱신한다
- 커밋 메시지 형식: `chapter-XX: 작업 내용`
- `scripts/validate-progress.sh`가 Claude hook으로 커밋 전 자동 실행된다
